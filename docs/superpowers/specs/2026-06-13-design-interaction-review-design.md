# 设计与交互审查迭代方案

## 背景

本轮由设计经理与交互测试经理分别审查核心页面，再由 CEO agent 终审。
审查未发现阻断使用的 P0，但确认择饮、探寻、学院、搜索与收藏存在信息承诺
不准确、状态连续性不足和操作绕路问题。

本方案只使用仓库现有 BJCP 风格、市场扩展风格、学院文章与本地收藏数据，
不新增外部事实，不改变风格数据、编号、参数或分类边界。

## 目标

1. 用户看到的推荐数量、场景状态与推荐区域保持一致。
2. 用户连续微调时不丢失正在阅读的解释状态。
3. BJCP 官方标准风格与市场扩展风格的身份表达始终清晰。
4. 学院筛选、搜索反馈与收藏操作减少歧义和绕路。
5. production 发布画像继续完全排除 beta AI 运行面。

## 批准范围

### 1. 择饮结果准确性

- 顶部数量只表达首屏实际展示的推荐数量。
- 首选和两个相关风格组成最多三个可见推荐。
- “全部复合匹配”排除已经展示的推荐，避免同一风格重复出现。
- 市场扩展风格使用“扩展风格”文字，不使用面向用户的 `EX`。

### 2. 择饮状态连续性

- 用户改变场景时，口味恢复该场景预设，解释区域收起，图表回到第一个视图。
- 用户只改变口味时，场景显示为“场景名 · 已微调”。
- 口味微调保留解释区域展开状态与当前图表索引。
- 推荐理由最多显示两行，不再被单行截断。

### 3. 图表语义

- 删除不随数据变化的雷达填充色块和星图装饰连线。
- 保留由当前推荐数据生成的指标、标签和节点。
- 不新增精确数值或新的推荐算法解释。

### 4. 探寻快速入口

- BJCP 分区只展示 BJCP IPA 快速入口。
- 市场扩展分区只展示市场扩展 IPA 快速入口。
- 快速入口标题明确当前体系。

### 5. 学院信息层级

- 页面使用模型中已有的“学院”标题与副标题。
- `simulator` 显示为“互动”，`tool` 保持“工具”，筛选标签不重名。
- 文章标题允许最多两行，避免长标题过早截断。

### 6. 搜索反馈

- 有结果时显示当前结果数量。
- 点击空结果的“清空重新搜索”后恢复输入焦点。
- 不改变搜索数据、匹配算法与结果排序。

### 7. 收藏可信交互

- 收藏页提供独立“取消”按钮，并阻止该操作触发详情跳转。
- 只有原子删除 API 返回成功后才移除列表项。
- 删除失败保留原列表并显示失败反馈。
- 收藏读取失败显示错误状态与重试入口，不伪装成“0 个收藏”。
- 空收藏状态与读取失败状态必须互斥。

## 视觉约束

- 沿用现有深色背景、橙色主强调和青色辅助强调。
- 不进行页面骨架重构，不新增插图、图标体系或卡片家族。
- 核心按钮触控高度不低于现有主要操作控件；新增取消与重试按钮沿用现有
  pill 操作样式。
- 辅助文案只做必要的字号和对比度调整，不扩大纯日期、编号等低优先级信息。

## 错误处理

- 收藏读取异常必须保留错误状态，重试后再决定展示列表或空状态。
- 收藏删除异常不得更新页面列表，不得上报成功事件。
- 不使用推测状态覆盖本地存储状态。

## 验收

- 自动化测试覆盖择饮排重、微调状态、视图保留、搜索反馈、学院筛选标签、
  收藏删除成功/失败和收藏读取失败。
- `npm test`、`npm run check:generated`、`npm run check:profiles` 全部通过。
- production 发布画像中不包含 `agent`、`skills/`、`aiDetail/` 或 `ai-entry`。
- 本轮无法取得微信开发者工具真实窗口截图，因此不得声称完成视觉真机验收；
  开发者工具与真机检查保留为正式发布前阻断项。

## 延后

- 不重构择饮整体信息架构。
- 不实现学院测验答题状态。
- 不开展全局无障碍语义改造。
- 不处理 beta AI 浮钮位置；production 已物理排除该组件，beta 布局另行审查。
