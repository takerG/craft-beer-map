A. 原生混合版

用 WXML/WXSS 做首页、列表和详情；用 Canvas 只承载大类内的局部地图。性能稳，后续发布到微信最直接。

B. 地图优先混合版

首页就进入 Canvas 地图，卡片和详情作为抽屉。视觉最接近原站，但小程序首屏、触屏缩放和低端机性能风险更高。

适合长期维护多端，但本次重构成本最高。

C. 跨端框架版

引入 Taro 或 uni-app，用组件模型统一 Web 和小程序。适合未来多端，但会改变项目技术栈和构建方式。
